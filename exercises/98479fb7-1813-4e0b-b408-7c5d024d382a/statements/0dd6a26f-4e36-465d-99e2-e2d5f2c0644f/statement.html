Proyecciones

De forma predeterminada, MongoDB devuelve todos los campos para todos los documentos coincidentes, como se ve en todos los resultados anteriores.

Las proyecciones reducen la sobrecarga de la red y los requisitos de procesamiento al limitar los campos que se devuelven en los documentos de resultados.

Las proyecciones se pasan como el segundo argumento del método find().

Debemos usar <campo>: 1 para incluir ese campo en el documento de resultados y/o usar <campo>: 0 para excluir ese campo. Debemos tener en cuenta que los valores 1 y 0 o true y false representan la inclusión y la exclusión en el documento resultante, respectivamente.

db.<collection>.find({<campo1>: <valor1><campo2>: <valor2>}, {<campo1>: [1|0], <campo2>: [1|0]})

Ejemplo

Como en el ejemplo del ejercicio anterior, muestra el name de las tarjetas de type 'rock' tienen un valor para hp de 123 y el de abilities de Refrigerate.


db.cards.find(
    {"hp":123, "type1":"rock", "abilities": "Refrigerate"}, 
    {"name":1})

Nota: Si queremos que no se muestre el campo _id, debemos añadir a la proyección "_id": 0

Ejercicio

Muestra el título (title) y el reparto (cast) de las películas de la colección movies_2015 que tienen un rated de TV-14 y un type igual a series
